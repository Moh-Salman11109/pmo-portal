Section 1

Dashboard Overview

The GRC Risk Intelligence Dashboard is an integrated platform for monitoring Key Risk Indicators (KRIs), the Risk Register, and Risk Appetite. It provides GRC Admins with a real-time, comprehensive view of organisational risks — all manageable directly from the portal.

📊

KRI Monitoring

8 Key Risk Indicators tracked with a RAG status system (Red / Amber / Green).

⚠️

Risk Register

A comprehensive list of all open risks with likelihood, impact scores, and mitigation actions.

🎯

Risk Appetite

Compares actual exposure against the maximum tolerable limit per risk category.

📈

Trend Chart

A line chart showing the last 12 readings for any selected KRI.

Section 3

Dashboard Components

1️⃣

KPI Strip (5 cards)

Displays total KRIs, Red/Amber/Green counts, and escalations required. Auto-refreshes when you press Refresh.

2️⃣

KRI Status Board

A table showing every active KRI with its latest value, RAG status, trend, and period. Click any row to open its trend chart.

3️⃣

Trend Chart

A line chart of the last 12 readings with threshold reference lines (Green / Amber / Red). Appears when a KRI row is selected.

4️⃣

Risk Appetite

Progress bar per category comparing actual exposure against the maximum tolerable score, with the appetite statement.

5️⃣

Top Risks

Top 6 open risks ranked by score (Likelihood × Impact). In Global Edit mode, all risks are shown.

🔄

Refresh Button

Reloads all data from SharePoint in real time. Use it after any data change to see the latest state.

Section 4

Adding a KRI Reading

A KRI reading records the actual value of an indicator for a specific time period. A new reading should be added monthly for every active KRI to keep the dashboard accurate.

1

Locate the KRI in the table

Open the GRC Dashboard and find the indicator you want to update in the KRI Status Board.

2

Click the + Reading button

The button appears in the last column of each row and is visible to GRC Admin only.

3

Fill in the reading form

Enter the actual value, period, RAG status, trend direction, and any comments. See the field reference table below.

4

Click Save Reading

The entry is written directly to SharePoint and the table and chart update automatically.

📋 KRI Reading Form — Field Reference
FieldDescriptionTypeRequired?
Actual ValueThe real measured value of the indicator for this periodNumber● Required
Previous ValueLast period's value for comparison (optional)Number○ Optional
PeriodMonth and year, e.g. 2026-05Month / Year● Required
RAG StatusGreen / Amber / Red based on value vs thresholdsChoice● Required
TrendImproving / Stable / WorseningChoice● Required
CommentsNotes or explanation for the readingText○ Optional
Escalation RequiredDoes this reading require escalation to senior management?Checkbox○ Optional
⚠️
Note: Set RAG Status manually by comparing the actual value against the KRI thresholds. You can view the thresholds by clicking on the KRI row to expand its trend chart.

Section 5

Global Edit Mode

The Global Edit button in the top-right of the header activates full edit mode — enabling add, edit, and delete operations across all dashboard data.

🚨
Warning: Delete operations cannot be undone. Always verify the data before confirming a deletion.
🔓 What becomes available in Global Edit mode
📊
Edit KRI Definition
Modify KRI name, category, measurement unit, thresholds (Green/Amber/Red), and threshold direction.
⚠️
Add New Risk
Create a new risk entry in the register with full details including scores, status, and mitigation summary.
✏️
Edit / Delete a Risk
Update likelihood, impact, status, appetite breach flag, and mitigation — or permanently remove the risk.
🎯
Add / Edit / Delete Appetite
Manage risk appetite entries per category including the appetite statement and exposure/limit scores.
1

Activate Edit Mode

Click the ✎ Global Edit button at the top right. It will turn green and show ✓ Edit Mode ON.

2

Additional buttons appear throughout the dashboard

Each KRI row gains an Edit KRI button. Each risk card gains Edit and Delete. + Add Risk and + Add Appetite Category buttons appear at the bottom of their sections.

3

Make your changes

Click the relevant button, fill in the form, and press Save. Changes are written directly to SharePoint.

4

Exit Edit Mode

Click the button again to deactivate Global Edit and return to the standard read/add view.

Risk Score = Likelihood × Impact. The score ranges from 1 to 25.

1 – 8
Low
Acceptable risk — monitor periodically
9 – 14
Medium
Requires an active mitigation plan
15 – 25
High
Requires immediate escalation

Section 7

Managing Risk Appetite

Risk Appetite defines the maximum tolerable level of exposure the organisation is willing to accept per risk category. It should be reviewed and updated whenever policy changes or exposure materially shifts.

📋 Risk Appetite Form — Field Reference
FieldDescriptionRequired?
Current Exposure ScoreThe actual current exposure level for this category● Required
Appetite StatusWithin Appetite / Near Limit / Breached● Required
Appetite StatementA sentence describing the organisation's stance (e.g. "We accept calculated strategic risks…")○ When adding new
Max Tolerable ScoreThe upper limit before the category is considered breached○ When adding new
💡
Tip: The progress bar auto-calculates: (Current Exposure ÷ Max Tolerable) × 100. Update Current Exposure Score monthly to keep the visual accurate for executive viewers.
